System Design: PS01 — Document Verification Pipeline

Full architecture overview — monolith pipeline on Polygon. Review each layer below.

System Flow
CITIZEN  →  [React Upload UI]  →  POST /api/verify  →  [FastAPI Server]
                                                  ↓
                             ① Compress  (Pillow / PyMuPDF)
                                                  ↓
                             ② OCR      (Google Vision API)
                                                  ↓
                             ③ Extract Fields  (regex + doc-type rules)
                                                  ↓
                             ④ Cross-Validate  (name / DOB / district)
                                                  ↓
                             ⑤ Blockchain   (web3.py → Polygon)
                                                  ↓
OFFICER  ←  [React Dashboard]  ←  JSON + tx_hash
Pipeline Stages

① Compress

  • JPEG/PNG → Pillow resize + quality
  • PDF → PyMuPDF render → recompress
  • Target: <300KB, preserve text clarity
  • Returns: compressed bytes + size delta

② OCR

  • Google Vision DOCUMENT_TEXT_DETECTION
  • Returns full text + bounding boxes
  • Hindi + English mixed script support
  • Per-word confidence scores

③ Extract Fields

  • Doc-type classifier (keyword matching)
  • Per-doc regex extractors:
  •   Aadhaar: name, DOB, gender, addr, UID
  •   PAN: name, DOB, PAN no, father name
  •   Income/Domicile/Caste: name, district

④ Cross-Validate

  • Name match across all docs
  • DOB match (Aadhaar ↔ PAN)
  • District match (domicile ↔ income cert)
  • Returns: MATCH / MISMATCH / MISSING
  • Overall confidence score (0–100)

⑤ Blockchain (Polygon Mumbai testnet)

Smart Contract stores
  • applicant_id
  • SHA-256 hash of each document
  • overall_confidence score
  • is_verified (bool)
  • timestamp
Returns to client
  • tx_hash (Polygonscan link)
  • block_number
  • contract_address
  • Immutable proof of verification
